Algumas boas práticas ao utilizar Git e Github:

1- criar uma branch nova antes de começar a trabalhar
2- seguir regras de nomenclatura pro nome da branch
    - Exemplos: quando é uma coisa nova sendo adicionada a branch chama: feature/sistema-de-cadastro
    - Quando conserta bugs: fix/resolve-bug-do-sistema-de-cadastro
    - Arquitetura do projeto: chore/muda-versão-kubernets
3- Pull requests pequenos e assertivos
    - Separar cada mudança em um pull request diferente 
4- Evitar subir arquivos desnecessários e privados
5- Comunicação aberta com os colegas
